# 零基础看懂 GitHub 仓库

这是一份面向零基础读者的 GitHub 仓库阅读学习资料。

它的目标不是教你立刻写代码，而是先帮你看懂一个仓库里常见的文件、目录、后缀、配置、依赖、命令、Git 状态和开发计划里的基础概念。

## 资料入口

| 入口 | 说明 |
|---|---|
| `docs/零基础看懂任何GitHub仓库学习资料.md` | 总版 Markdown，适合从头到尾系统阅读 |
| `sections/` | 分章版 Markdown，适合按主题单独阅读和维护 |
| `sections/README.md` | 分章目录 |

## 适合谁

- 没有编程基础，但需要看懂 GitHub 仓库的人。
- 使用 Codex、VS Code、GitHub、AI 工具时，经常被文件名、后缀、命令和配置卡住的人。
- 想先建立“仓库由哪些稳定部分组成”的整体理解，再逐步学习代码细节的人。

## 这份资料覆盖什么

- GitHub 仓库的基本组成。
- README、docs、src、app、server、tests、scripts、tools 等常见目录。
- `.md`、`.json`、`.toml`、`.yml`、`.env`、`.mjs`、`.csv`、`.hbs` 等常见后缀。
- Node.js、JavaScript、TypeScript、npm、Python、Docker、Jupyter、Nginx、MCP 等基础概念。
- VS Code Explorer 里的颜色、字母和 Git 状态。
- 如何把陌生文件、目录、命令先归类，再继续理解。

## 使用方式

先读总版 Markdown。遇到某个主题想单独复习时，再去 `sections/` 里找对应章节。

读仓库时，不要先要求自己看懂每一行代码。先判断：

1. 这个东西是文件、文件夹、命令、字段，还是状态提示。
2. 它属于说明区、代码区、配置区、依赖区、测试区、部署区、协作区，还是数据区。
3. 它和哪个工具有关：Git、Node/npm、Python、Docker、VS Code、GitHub Actions，还是 AI/MCP。

这样读仓库会更稳，不会被单个生词卡住。
